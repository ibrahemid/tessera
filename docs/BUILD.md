# Building & testing Tessera

## Go CLI (no Xcode needed)

```sh
cd go
go build ./...
go vet ./...
go test -race ./...
go build -o tess ./cmd/tess      # or: go install ./cmd/tess
```

## Swift core — local verification without Xcode

Full Xcode is NOT required to verify the Swift core against the shared vectors.
The interop verifier compiles `TesseraCore` with `swiftc` and checks every shared
vector (OTP, base32, otpauth, migration, canonical JSON, XChaCha20-Poly1305):

```sh
swiftc -O swift/Sources/TesseraCore/*.swift swift/Tools/verify/main.swift -o /tmp/tessera_verify
/tmp/tessera_verify "$(pwd)/spec"
```

Expected: `ALL SWIFT INTEROP CHECKS PASSED`.

## Swift package tests (argon2id + full vault) — needs SwiftPM

`argon2id` is the one primitive CryptoKit lacks, so the passphrase-wrap and full
Go→Swift envelope cross-decrypt are verified via SwiftPM/XCTest (which pulls the
Argon2Swift dependency). This requires full Xcode (the Command Line Tools alone
ship a broken SwiftPM):

```sh
cd swift
swift test
```

## The macOS app — needs full Xcode

The menu-bar app uses SwiftUI, MenuBarExtra, Keychain/Secure Enclave,
ScreenCaptureKit, and ServiceManagement, and must be built and signed with Xcode.

```sh
brew install xcodegen          # one-time
cd swift
xcodegen generate              # creates Tessera.xcodeproj from project.yml
open Tessera.xcodeproj
```

Set `DEVELOPMENT_TEAM` (your Apple Developer team ID) in the target's signing
settings, then Run. See `docs/APP_STORE.md` for distribution.

## Regenerating interop vectors

The Steam/argon2id/AEAD/vault vectors are generated by `go/internal/vectorgen`
and pinned into `spec/testvectors.json` (the RFC vectors are external ground
truth and hand-authored). To regenerate:

```sh
go -C go run ./internal/vectorgen            # prints snippets to paste
go -C go run ./internal/vectorgen edge > spec/canonical_edge.json
```
