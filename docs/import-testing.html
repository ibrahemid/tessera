2Get the apps to migrate from

2FAS iOS, Android · open source
download / site · source
Export: Settings → Backup → Export. If it offers a password, leave it OFF. Saves a .2fas file (it's JSON).
Encrypted only if you set a backup password — leave it off for the plain importer.
Aegis Android · open source
download / site · source
Export: Settings → Import & Export → Export. Untick “Encrypt the vault”. Saves a .json file.
Encrypted by default — untick encryption when exporting.
Raivo iOS, macOS · open source
download / site · source
Export: Settings → Export → JSON. You get a .json (sometimes inside a .zip — unzip it).
Plain JSON — works as-is.
Ente Auth iOS, Android, desktop · open source
download / site · source
Export: Settings → Data → Export → plain text. Saves a .txt of otpauth:// links.
Plain otpauth links — import with tess import --file.
Google Authenticator iOS, Android
download / site
Export: ⋮ → Transfer accounts → Export. It shows QR code(s) on screen — no file. In Tessera use Add → Scan, or screenshot and “Read QR from an image”.
No file export — use the on-screen QR path.
Authy iOS, Android
download / site
Export: No usable export (the desktop app that allowed it is retired). Skip unless you already have a backup.
Not exportable — listed for completeness.

3Export from each app, then send me the file

  1. In the app, add the sample accounts (step 1), then run its Export (step 2).
  2. Export unencrypted where there's a choice — those import right now (Aegis, 2FAS, Raivo, Ente).
  3. Drop the files in the repo at docs/import-samples/ and tell me which app each came from.
  4. Want encrypted-export support too? Also send one encrypted export per app (with the password you used) and I'll add decryption.

What works today

Tessera already imports: otpauth links, Google Authenticator QR exports, QR images/on-screen, and unencrypted Aegis / 2FAS / Raivo / Ente files (drag the file into Add → or tess import --file export.json). Encrypted app exports are detected and tell you to export unencrypted — full decryption is the next step.
